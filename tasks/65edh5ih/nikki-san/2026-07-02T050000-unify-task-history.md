## 目的

タスク履歴の運用が各リポジトリで別々に育っていた（ファイル名の不統一・アーカイブの数え方の違い）。
全リポジトリ共通の規約を `docs/task-history.md` として配布したので、このリポジトリの履歴を規約へ移行する。

## 作業内容

1. 配布済みの規約 **`docs/task-history.md`** を読む（このタスクと同じ同期で届いている）。
2. 現在の履歴ファイル（`AGENTS.md` の固有パートが指しているもの。名前・場所はこのリポジトリの
   現状に従って自分で特定する）を **`docs/AI_TASK_HISTORY.md`** へ移行する:
   - `git mv` でリネーム（履歴エントリの中身は温存。書き直さない）。
   - 各エントリの見出しを `## YYYY-MM-DD 短いタイトル` 形式に揃える（日付が無いものは
     git log から推定してよい）。新しいエントリが上。
3. 規約の保持ルールを適用する: エントリのある日付で数えて**直近5日分**を残し、
   それより古い日付のエントリを **`docs/history-archive/<YYYY>.md`** の先頭へ移す。
4. 旧ファイル名への参照（`AGENTS.md` の固有パート・フック・スクリプト等）をすべて新パスに更新する。
   固有パートに書かれていた履歴の書式・保持件数などの独自ルールは削除する（規約 doc が正。
   固有パートに残すのは規約からの逸脱がある場合のみ）。
5. 完了したら `.ai-ops/outbox/` に次の提案を置いて main に載せる（書式の詳細: `docs/outbox-proposal.md`）:

   ```markdown
   ---
   種別: task-done
   対象ファイル: 2026-07-02T050000-unify-task-history.md
   理由: 履歴を docs/AI_TASK_HISTORY.md へ統一し、5日分保持＋アーカイブを適用
   ---
   ```

## 完了条件

- `docs/AI_TASK_HISTORY.md` が規約どおりの体裁で存在し、旧ファイルが残っていない。
- リポジトリ内に旧ファイル名への参照が残っていない（grep で確認）。
- 直近5日分を超える古いエントリが `docs/history-archive/<YYYY>.md` に移っている（超過が無ければ不要）。
- `task-done` 提案が main に載っている。
