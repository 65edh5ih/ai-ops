# リポジトリ固有の手順系 doc を SOP 書式規約に準拠させる

- **目的**: 全リポジトリで手順 doc（SOP）の書式規約 `docs/sop-format.md` の運用を開始した
  （トリガの明示・パラメータ化・ステップごとの完了条件・制約強度 MUST/SHOULD/MAY）。共通 doc は
  ai-ops 側で順次準拠させるが、このリポジトリ**固有**の手順 doc はこのリポジトリでしか直せないため依頼する。
  同種の依頼を nikki-san にも出し、そちらは対応済み。
- **作業内容**:
  1. `docs/` 配下（および README 等に埋め込まれた手順節）から「エージェントに実行させる手順」を
     書いた doc を洗い出す。
  2. 各 doc を `docs/sop-format.md` の書式に準拠させる（適用外: 設計 doc・タスク履歴・README 本文。
     手順を含まない文書には適用しない。判断に迷うものは変換せず一覧に残すだけでよい）。
  3. 書式の変換のみで、**手順の内容は変えない**。手順が実態と違うと気づいた箇所は、確証がある
     ものだけ直し、不確かなものは doc 内に TODO として残す。
- **完了条件**: 固有の手順 doc すべてについて、(a) 準拠済み、または (b) 適用外・保留と判断した理由付き
  一覧がタスク履歴（`docs/AI_TASK_HISTORY.md`）に記録されていること。完了したら outbox の
  `種別: task-done` 提案でこのタスクの消化を報告する（→ `docs/outbox-proposal.md`）。
