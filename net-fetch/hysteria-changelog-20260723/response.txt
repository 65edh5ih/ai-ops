  <h1>Changelog</h1>

<h2 id="2100">2.10.0</h2>
<ul>
<li>Added <a href="../advanced/ECH/">Encrypted Client Hello</a> support</li>
<li>Added a <code>bandwidth.disableLossCompensation</code> option for disabling Brutal congestion control's loss compensation mechanism, which can improve stability in some cases</li>
<li>Various dependency updates and minor bug fixes</li>
</ul>
<h2 id="293">2.9.3</h2>
<ul>
<li>Added <a href="../advanced/Realms/">Hysteria Realms</a> UPnP/NAT-PMP port mapping support for easier NAT traversal</li>
<li>Added a Hysteria Realms <code>ipMode</code> option to restrict connections to IPv4 or IPv6 only</li>
<li>Updated quic-go to v0.60.0</li>
<li>Fixed <code>+</code> in the authentication credentials being corrupted to a space when parsing <code>hysteria2://</code> sharing links</li>
<li>Fixed the HTTP proxy rejecting valid Basic authentication credentials</li>
<li>Fixed the HTTPS proxy outbound using <code>host:port</code> instead of the hostname as the TLS SNI</li>
<li>Fixed a data race in the client's HTTP/SOCKS5 listener multiplexer</li>
<li>Fixed nftables redirect failure when the server binds to a specific IPv6 address</li>
<li>DNS resolver now limits CNAME chain length to prevent unbounded recursion</li>
</ul>
<h2 id="292">2.9.2</h2>
<blockquote>
<p>This release contains important security fixes and we strongly encourage everyone to upgrade.</p>
</blockquote>
<ul>
<li>Added <a href="../advanced/Full-Server-Config/#obfuscation">Gecko obfuscation</a>: a new experimental obfuscation layer that fragments QUIC handshake packets</li>
<li>Fixed a security issue where UDP packets could bypass ACL</li>
<li>Fixed a potential server OOM caused by incomplete or oversized HTTP requests during sniff</li>
<li>Fixed an ACL bypass via trailing dots in domain names (e.g. <code>example.com.</code>)</li>
<li>Fixed incorrect destination handling in the SOCKS5 UDP outbound</li>
<li>Fixed <code>https://</code> prefix detection in the DoH resolver</li>
<li>Outbound rules now reject invalid port values</li>
<li>Minor performance improvements</li>
</ul>
<h2 id="291">2.9.1</h2>
<ul>
<li>Fixed an issue where the client failed to connect to a server behind symmetric NAT, improving hole punching success rate</li>
<li>The <code>hysteria cert</code> command now includes <code>sniGuard: disable</code> in its sample server config, so self-signed certs work with Realms out of the box</li>
</ul>
<h2 id="290">2.9.0</h2>
<ul>
<li>Added <a href="../advanced/Realms/">Hysteria Realms</a>: No public IP? No problem. Punches through your NAT to let you host a Hysteria server from home, a cellular network, or even a coffee shop. Clients connect directly, peer-to-peer. No port forwarding, no relay.</li>
</ul>
<h2 id="282">2.8.2</h2>
<blockquote>
<p>This release contains important security fixes and we strongly encourage everyone to upgrade.</p>
</blockquote>
<ul>
<li>Fixed a security issue where, when sniff was enabled, an attacker could craft malicious QUIC packets to cause a server OOM crash</li>
<li>Fixed a compatibility issue with some older versions of nftables when server port hopping was enabled</li>
<li>Fixed a potential thread safety issue in salamander obfs</li>
<li><strong>Important: Due to changes in how some QUIC handshake parameters are handled, UDP forwarding will not work when v2.8.2 clients connect to older servers (TCP is unaffected). New servers are fully compatible with both new and old clients. We strongly recommend upgrading both servers and clients.</strong></li>
</ul>
<h2 id="281">2.8.1</h2>
<blockquote>
<p>This release contains important fixes and we strongly encourage everyone to upgrade.</p>
</blockquote>
<ul>
<li>Fixed an issue where client connections could cause the server to crash when using BBR/Reno as the congestion control algorithm</li>
<li>Fixed iptables calls potentially failing due to lock contention during server port-range listening</li>
<li>Added <code>HYSTERIA_FIREWALL_BACKEND</code> environment variable to specify the firewall backend (<code>iptables</code> or <code>nftables</code>) for server port-range listening</li>
</ul>
<h2 id="280">2.8.0</h2>
<ul>
<li>Added <a href="../advanced/Full-Server-Config/#congestion">configurable congestion control</a>: select between BBR and Reno, with three BBR profiles (<code>standard</code>, <code>conservative</code>, <code>aggressive</code>) for fine-tuning congestion control behavior</li>
<li>Added <a href="../advanced/Port-Hopping/#built-in-port-range-linux">server-side UDP port range listening</a> (Linux only): the server can now listen on a port range and automatically set up nftables/iptables redirect rules</li>
<li>Added <a href="../advanced/Port-Hopping/#client">random port hopping interval</a>: use <code>minHopInterval</code>/<code>maxHopInterval</code> for a randomized hopping pattern instead of a fixed interval</li>
<li>Added <code>xForwarded</code> option to <a href="../advanced/Full-Server-Config/#masquerade">masquerade proxy</a> for setting <code>X-Forwarded-For</code>/<code>Host</code>/<code>Proto</code> headers</li>
<li>Minor BBR fixes and improvements</li>
<li>Minor port hopping fixes and improvements</li>
</ul>
<h2 id="271">2.7.1</h2>
<ul>
<li>Updated quic-go to v0.59.0, QUIC protocol level improvements</li>
<li>Speed test now defaults to time-based mode, testing download and upload for 10 seconds each. Use <code>--duration</code> to customize. The previous size-based behavior is still available via <code>--data-size</code>.</li>
<li>Minor code cleanup</li>
</ul>
<h2 id="270">2.7.0</h2>
<blockquote>
<p>This release contains important fixes and we strongly encourage everyone to upgrade.</p>
</blockquote>
<ul>
<li>Updated quic-go to v0.57.1</li>
<li>Fixed a long-standing bug in BBR that caused the connection to send faster than the available bandwidth in some cases</li>
<li>Minor performance improvements</li>
</ul>
<h2 id="265">2.6.5</h2>
<blockquote>
<p>This release contains important fixes and we strongly encourage everyone to upgrade.</p>
</blockquote>
<ul>
<li>Fixed a server-side memory leak issue that accumulates with each client connection</li>
</ul>
<h2 id="264">2.6.4</h2>
<blockquote>
<p>This release contains important fixes and we strongly encourage everyone to upgrade.</p>
</blockquote>
<ul>
<li>Security fix &amp; behavior change: <code>tls.pinSHA256</code> now matches only the fingerprint of the leaf certificate, instead of any certificate in the chain. This change mitigates MITM risks in cases where <code>insecure=true</code> by preventing 1) user accidentally pinning a CA certificate, which would allow any certificate issued by that CA to be accepted, and 2) attacker constructing a forged certificate chain by combining their own leaf certificate with the user server's certificate.</li>
<li>Fix tun mode UDP packet AF corruption</li>
<li>Updated quic-go to v0.54.0</li>
</ul>
<h2 id="263">2.6.3</h2>
<ul>
<li>Added mTLS support for client certificate authentication</li>
<li>Fixed a memory leak issue in tun mode</li>
<li>Fixed an issue where DNS resolution failed in tun mode on Linux systems using systemd-resolved</li>
<li>Fixed a bug in the ACL cache that caused rules with different ports or protocols to be applied to irrelevant connections</li>
<li>Removed the license-conflicted DoH library and replaced it with an in-house implementation</li>
<li>Fixed a race condition in UDP session handling</li>
</ul>
<h2 id="262">2.6.2</h2>
<ul>
<li>Updated quic-go to v0.52.0</li>
<li>ClientHello is now fragmented during TLS handshake, which can bypass some firewalls' SNI filtering.</li>
<li>The <code>--qr</code> option in client mode is deprecated; use the <code>share</code> subcommand to generate links and QR codes instead.</li>
</ul>
<h2 id="261">2.6.1</h2>
<ul>
<li>The server's direct outbound now supports TCP Fast Open</li>
<li>Changed <code>LibVersion</code> to <code>Libraries</code> in the output of the <code>version</code> subcommand</li>
<li>Added support to disable TLS verification (<code>insecure</code>) for the masquerade proxy website</li>
<li>Fixed issues with tun not functioning on Linux when <code>ipv6.disable=1</code> is set</li>
<li>Added support for <code>LoongArch64</code></li>
<li>Updated quic-go to version 0.49.0</li>
<li>Made the username in <code>userpass</code> authentication case-insensitive</li>
</ul>
<h2 id="260">2.6.0</h2>
<blockquote>
<p>This release contains important fixes and we strongly encourage everyone to upgrade.</p>
</blockquote>
<ul>
<li>Fixed a bug where the client would freeze on startup if the port hopping range contained 65535</li>
<li>Added a new <code>/dump/streams</code> endpoint to the traffic stats API for retrieving information on active QUIC streams</li>
<li>Updated quic-go to v0.48.1</li>
<li>The <code>version</code> subcommand now includes the toolchain &amp; quic-go version information</li>
<li>Added a new <code>share</code> subcommand to the client for generating sharing links &amp; QR codes</li>
<li>The server now validates the <code>masquerade.proxy.url</code> field to ensure it's a valid HTTP or HTTPS URL</li>
</ul>
<h2 id="252">2.5.2</h2>
<ul>
<li>By default, the server now terminates the TLS handshake if the SNI sent by a client does not match the certificate. A new <code>sniGuard</code> option has been added to the <code>tls</code> section to control this behavior</li>
<li>Fixed the issue where sniffing could not recognize fragmented QUIC packets</li>
<li>Updated quic-go to v0.47.0</li>
</ul>
<h2 id="251">2.5.1</h2>
<ul>
<li>Fixed a bug in HTTP sniffing that caused all connections to non-standard ports (non-80) to fail</li>
<li>Fixed a bug in the client where the authentication password was not escaped when parsing sharing links</li>
<li>Updated quic-go to v0.46.0</li>
</ul>
<h2 id="250">2.5.0</h2>
<ul>
<li>Added support for ACME DNS challenge, including several common providers such as Cloudflare, GoDaddy, and Name.com</li>
<li>Added server-side protocol sniffing, supporting HTTP, TLS (HTTPS), and QUIC (HTTP/3)</li>
<li>Fixed the issue with inaccurate unit conversion in the speed test command (1024 -&gt; 1000)</li>
</ul>
<h2 id="245">2.4.5</h2>
<ul>
<li>Fixed some logic issues in BBR, and added <code>HYSTERIA_BBR_DEBUG</code> environment variable for printing debug information</li>
<li>Fixed compatibility issues of the HTTP proxy with certain programs</li>
<li>Updated quic-go to v0.44.0</li>
</ul>
<h2 id="244">2.4.4</h2>
<blockquote>
<p>This release contains important fixes and we strongly encourage everyone to upgrade.</p>
</blockquote>
<ul>
<li>Fixed a memory leak in quic-go between 2.4.2 and 2.4.3</li>
<li>Added a new GET <code>/online</code> endpoint to the traffic stats API for retrieving current online users and their connection counts</li>
<li>The client now gracefully closes the QUIC connection when exiting</li>
</ul>
<h2 id="243">2.4.3</h2>
<blockquote>
<p>This release contains important fixes and we strongly encourage everyone to upgrade.</p>
</blockquote>
<ul>
<li>Fixed a bug introduced in the previous version that caused UDP forwarding to not work properly</li>
</ul>
<h2 id="242">2.4.2</h2>
<ul>
<li>Small tweaks to Brutal congestion control to improve performance at high speeds</li>
<li>When using a local certificate, the server now checks if the files can be accessed on startup. This ensures that access problems are detected immediately, rather than when accepting client connections.</li>
<li>Updated quic-go to v0.43.0</li>
</ul>
<h2 id="241">2.4.1</h2>
<ul>
<li>The client now supports listening to both HTTP and SOCKS5 on a single port by making their <code>listen</code> addresses the same</li>
<li>The client has added a new <code>sockopts</code> section to <code>quic</code> which allows the user to specify the bind interface, fwmark and FD control socket path for outbound QUIC connections. This is a feature mainly for Android app development.</li>
</ul>
<h2 id="240">2.4.0</h2>
<ul>
<li>Added TUN mode to client (supports Windows, Linux &amp; macOS)</li>
<li>ACL now supports matching port ranges (e.g. <code>reject(all, udp/40000-50000)</code>)</li>
<li>Added a <code>listenHost</code> field to the server ACME config to allow specifying the listening address for receiving validation requests</li>
<li>Updated quic-go to v0.42.0</li>
<li>Lowered the log level for proxied connection errors from error to warning</li>
</ul>
<h2 id="230">2.3.0</h2>
<ul>
<li>Added a built-in speed test subcommand for client &amp; its server-side support</li>
<li>Automatically try to re-download GeoIP/GeoSite DB files if they fail to load</li>
<li>Better SOCKS5 outbound error messages</li>
<li>Fixed a bug where the dual stack listen address was actually only listening for IPv6 on FreeBSD</li>
</ul>
<h2 id="224">2.2.4</h2>
<blockquote>
<p>This release contains important fixes and we strongly encourage everyone to upgrade.</p>
</blockquote>
<ul>
<li><strong>[Important]</strong> Fixed a bug where a connection timeout would block other connections from being established</li>
<li>Updated quic-go to v0.41.0</li>
</ul>
<h2 id="223">2.2.3</h2>
<ul>
<li>Fixed a bug where using an IPv4/IPv6 specific listening address like <code>0.0.0.0:443</code> or <code>[::]:443</code> would still result in listening on both IPv4/IPv6</li>
<li>Delay server address DNS resolution until connection attempt when lazy mode is enabled</li>
<li>Local TLS certificates are now read every time a TLS handshake takes place, allowing users to update files without restarting the server</li>
</ul>
<h2 id="222">2.2.2</h2>
<ul>
<li>Fixed a bug introduced in the previous version that broke the automatic reconnection of the client</li>
<li>Added <code>suffix:</code> support to ACL for matching a domain and all its subdomains (e.g. <code>reject(suffix:baidu.com)</code>)</li>
</ul>
<h2 id="221">2.2.1</h2>
<ul>
<li>Added GeoIP &amp; GeoSite auto update (<code>geoUpdateInterval</code> field under ACL, default is 1 week)</li>
<li>Client now shows handshake information after connecting to the server, currently includes UDP forwarding availability &amp; tx rate</li>
<li>Changed the basis for bandwidth conversion (Kbps/Mbps/Gbps/Tbps) from 1024 to 1000</li>
<li>Added RISC-V (riscv64) support</li>
<li>Updated quic-go to v0.40.0</li>
</ul>
<h2 id="220">2.2.0</h2>
<ul>
<li>Added GeoSite support to ACL (both GeoIP and GeoSite now use the v2ray "dat" format database)</li>
<li>Added support for non-English domains (IDN) to ACL (e.g. <code>v6_only(战狼*.中国)</code>)</li>
<li>Added WebSocket support to masquerade proxy mode</li>
<li>Added secret-based authentication to Traffic Stats API</li>
<li>Fixed compatibility issues on certain Linux systems</li>
</ul>
<h2 id="211">2.1.1</h2>
<blockquote>
<p>This release contains important fixes and we strongly encourage everyone to upgrade.</p>
</blockquote>
<ul>
<li>Fixed a bug where a specially crafted UDP message packet could cause the server to crash</li>
<li>Fixed compatibility issues on FreeBSD</li>
<li>Windows users can now launch directly by double-clicking the exe file</li>
</ul>
<h2 id="210">2.1.0</h2>
<ul>
<li>Fixed a memory leak in BBR</li>
<li>Minor tweaks to Brutal congestion control</li>
<li>Added string mode to masquerade</li>
<li>Added HTTP/HTTPS proxy outbound</li>
</ul>
<h2 id="204">2.0.4</h2>
<ul>
<li>Optimized and fixed some issues in Brutal CC</li>
<li>Fixed problem where BBR could freeze the connection and cause CPU usage to spike under certain conditions</li>
<li>Fixed two race condition issues</li>
<li>Added <code>HYSTERIA_BRUTAL_DEBUG</code> environment variable. When enabled, it prints information like current RTT, packet loss, MTU, etc.</li>
</ul>
<h2 id="203">2.0.3</h2>
<blockquote>
<p>This release contains important fixes and we strongly encourage everyone to upgrade.</p>
</blockquote>
<ul>
<li><strong>[Important]</strong> Fixed the problem where when using BBR (either the client doesn't set bandwidth or the server has <code>ignoreClientBandwidth</code> enabled), due to a bug in the BBR implementation, it could not accurately determine the bandwidth and send packets much faster than the limit.</li>
<li>Fixed the problem where ZeroSSL couldn't acquire certificates due to missing EAB.</li>
</ul>
<h2 id="202">2.0.2</h2>
<ul>
<li>Fixed connection issues on some devices due to lack of GSO support</li>
<li>Added HTTP/HTTPS (TCP) masquerade servers</li>
<li>Added Android builds</li>
</ul>
<h2 id="201">2.0.1</h2>
<ul>
<li>Added TCP redirect mode</li>
<li>Log HTTP requests handled by masquerade (debug level)</li>
<li>Added environment variable <code>HYSTERIA_ACME_DIR</code> to control ACME data directory location</li>
</ul>
<h2 id="200">2.0.0</h2>
<p>This is the first stable release of Hysteria 2. It's almost a complete rewrite of the original Hysteria, with a new protocol, new features, and various improvements.</p>
