# リポジトリ横断タスク（`.ai-ops/tasks/`）の使い方

「このリポジトリでやったのと同じ作業を別リポジトリでもやってほしい」等、**別リポジトリでの作業が
必要になったとき**の依頼経路。1セッション=1リポジトリの制約下で、依頼を ai-ops 経由で対象リポジトリの
手元（`.ai-ops/tasks/`）まで物理的に届ける。Notion 等の外部ツールへの転記は不要。

経路: 依頼元の `.ai-ops/outbox/`（`種別: task`）→ ai-ops の `tasks/<owner>/<repo>/` → sync で
対象リポジトリの `.ai-ops/tasks/` へ配布 → 対象リポジトリのセッションが実行 → `種別: task-done` で消化。
途中に取り込み PR（ai-ops）と同期 PR（対象）が挟まり、**オーナーが2回マージして初めて届く**＝届いている
タスクは承認済みとみなしてよい。

## 依頼する側（タスクを書くとき）

本文を読むのは**このセッションの文脈を何も持たない**、対象リポジトリのエージェント。次を必ず含める:

1. **目的（なぜ）**: 何を達成したいか。依頼元で何があったか。
2. **作業内容（何を）**: 具体的に。依頼元リポジトリのファイルは参照できないので、必要なコード・設定・
   手順は**本文に貼り込む**（「◯◯リポジトリの △△ を見て」は書かない——見えない）。
3. **完了条件**: 何ができたら終わりか。
4. 分かるなら対象リポジトリ側の関連パス・注意点。

依頼を outbox に置いたら（書式: [`docs/outbox-proposal.md`](outbox-proposal.md)）、**ユーザーに次を伝える**:

- 「ai-ops の取り込み PR → 対象リポジトリの同期 PR の順にマージされると届きます」
- **対象リポジトリのセッションに貼るコピペ用の指示文**（1つずつ・冒頭に1行の概要。Claude Code は
  最初の入力からセッションタイトルを生成するため）:

  ```
  <作業の1行概要>
  .ai-ops/tasks/<ファイル名> に依頼があります。読んで作業してください。
  ```

## 受ける側（`.ai-ops/tasks/` にファイルがあるとき）

- 現在のユーザー依頼がそのタスクを指しているなら、そのまま実行する。
- 別件の作業中に気づいたら、現在の依頼と両立する場合のみ対応し、そうでなければ存在をユーザーに
  伝えるに留める（勝手にセッションの目的を乗り換えない）。
- `.ai-ops/tasks/` のファイルは同期管理下（手で消しても次回 sync で復活する）。**完了したら
  `種別: task-done` 提案で消化する**（→ `docs/outbox-proposal.md`）。作業の「なぜ」は通常どおり
  このリポジトリの履歴ファイルに残す。

## ai-ops 側でできること

ai-ops のセッション（またはオーナー）は、`tasks/<owner>/<repo>/` にファイルを直接コミット／削除して
起票・消化できる（outbox を経由するのは consumer のセッションだけ）。ai-ops 自身への依頼はタスク機構を
使わず、ユーザーが ai-ops のセッションに直接指示する。
